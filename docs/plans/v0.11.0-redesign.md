# MyAgent v0.11.0+ 架构整改方案

> 基于 Hermes Agent、OpenClaw、OpenHarness、Claude Code 四方深度调研

---

## 一、现状诊断：为什么 MyAgent 只是 Chat Bot

### 1.1 核心差距对比

| 维度 | MyAgent 现状 | Claude Code | OpenHarness (ohmo) | Hermes Agent |
|------|-------------|-------------|-------------------|--------------|
| **工作空间** | `~/.myagent/` 只有 sessions/logs | `~/.claude/projects/<slug>/` 含代码+记忆 | `~/.ohmo/` 含 soul/user/memory/skills | Project 级 Workspace |
| **Memory** | 无结构化的 Markdown 列表 | **MEMORY.md 索引 + 分类记忆文件** | `memory/` 目录 + MEMORY.md 索引 | 项目上下文 + 用户画像 |
| **用户画像** | ❌ 无 | `user.md` + `feedback/` 自动收集 | `user.md` 模板引导填写 | User Profile |
| **Agent 身份** | 4 个静态定义（general/plan/worker/explore） | Agent 无独立身份，靠记忆区分 | `SOUL.md` + `IDENTITY.md` 定义 Agent 人格 | Agent Registry |
| **代码理解** | ❌ 无 | **自动扫描代码库生成索引** | ❌ 无 | Codebase Index |
| **任务管理** | 简单 Todo 工具 | Task 系统（Plan → Execute） | ❌ 无 | Task Manager |
| **多 Agent 协作** | SwarmCoordinator 框架无 UI | 单 Agent，靠记忆持久化 | ❌ 无 | Agent Teams |
| **初始化体验** | `myagent init` 配置 LLM+Gateway | **首次对话引导创建用户画像** | `initialize_workspace()` 生成模板文件 | Onboarding Wizard |

### 1.2 关键发现

**Claude Code 的记忆系统（最成熟）：**
- `MEMORY.md` 作为索引（200 行/25KB 上限），每个记忆独立文件
- 四种记忆类型：`user`（用户画像）、`feedback`（反馈）、`project`（项目）、`reference`（引用）
- Frontmatter 格式：`name`, `description`, `type`
- 自动收集：LLM 在对话中自动识别并保存记忆
- 记忆漂移处理：使用记忆前验证是否仍有效

**OpenHarness (ohmo) 的工作空间（最完整）：**
- `SOUL.md`：Agent 的灵魂/人格定义
- `user.md`：用户画像模板（引导式填写）
- `IDENTITY.md`：Agent 身份标识
- `BOOTSTRAP.md`：首次对话引导
- `memory/`：持久化记忆目录
- `skills/`：技能目录
- `state.json`：状态持久化

**Hermes Agent 的协作模式（最先进）：**
- Plan → Execute → Review 工作流
- Agent Teams：多个 Agent 并行协作
- 任务分配、追踪、依赖管理

## 三、迭代路线图（细化步骤）

### Phase 1: Workspace 基础（v0.11.0）
**目标：让 MyAgent 有"家"，不再是无状态的 Chat Bot**

#### 3.1.1 工作空间目录结构
```
~/.myagent/
├── soul.md              # Agent 灵魂（系统级）
├── user.md              # 用户画像（系统级）
├── identity.md          # Agent 身份（系统级）
├── memory/              # 系统级记忆
│   ├── MEMORY.md        # 记忆索引
│   └── *.md             # 分类记忆文件
├── skills/              # 技能目录
├── projects/            # 项目工作空间
│   └── default/         # 默认项目
│       ├── .myagent/    # 项目级配置
│       │   ├── memory/  # 项目级记忆
│       │   ├── agents/  # 项目级 Agent
│       │   └── tasks/   # 任务历史
│       └── ...          # 项目文件
├── sessions/            # 对话历史
├── logs/                # 日志
└── workspace/           # 通用工作区
```

#### 3.1.2 初始化流程改造
- `myagent init` 生成完整的 Workspace 模板文件
- 首次启动时引导用户填写 `user.md`
- 创建 `default` 项目作为默认工作空间

#### 3.1.3 模板文件设计

**soul.md**：
```markdown
# SOUL.md - Who You Are

You are MyAgent, an autonomous AI assistant.

## Core truths
- Be genuinely helpful, not performatively helpful
- Have judgment and explain your reasons
- Be resourceful before asking
- Earn trust through competence

## Boundaries
- Private things stay private
- When in doubt, ask before acting
```

**user.md**（引导式模板）：
```markdown
# user.md - About Your Human

## Profile
- Name:
- Timezone:
- Languages:

## Defaults
- Preferred tone:
- Decision style:

## Ongoing context
- Main projects:
- Tools and platforms:

## Preferences
- What they want more of:
- What tends to annoy them:
```

**MEMORY.md 索引格式**：
```markdown
# Memory Index

- [User Role](user_role.md) — 用户是后端工程师，专注 Python/Go
- [Feedback Style](feedback_style.md) — 喜欢简洁回答，不要总结
- [Project Context](project_context.md) — 正在开发 MyAgent 项目
```

**记忆文件格式**：
```markdown
---
name: User Role
description: User is a backend engineer focused on Python and Go
type: user
---

User is a senior backend engineer with 8 years of experience.
Primary languages: Python, Go.
Currently working on: MyAgent (AI agent platform).

**Why:** Understanding user's technical background helps tailor explanations.
**How to apply:** Use technical terms freely, suggest best practices from Python/Go ecosystem.
```

### Phase 2: Memory 系统（v0.12.0）
**目标：让 Agent 记住用户，越用越懂**

#### 3.2.1 记忆类型（参考 Claude Code）
- `user`：用户画像（角色、技能、偏好）
- `feedback`：反馈指导（该做什么、不该做什么）
- `project`：项目上下文（目标、决策、事件）
- `reference`：外部引用（文档、链接、系统）

#### 3.2.2 自动收集机制
- LLM 分析对话内容，自动识别值得记忆的信息
- 使用工具 `save_memory` 保存记忆
- 每次对话结束时自动总结并更新记忆

#### 3.2.3 记忆读取
- 每次对话开始时加载 MEMORY.md 索引
- 根据对话内容动态检索相关记忆
- 记忆注入到系统提示词中

#### 3.2.4 手动编辑
- Web UI 提供 Memory 管理页面
- 可以查看、编辑、删除记忆
- 支持手动添加新记忆

---

### Phase 3: Plan → Execute → Review（v0.13.0）
**目标：让 Agent 能分解任务、执行、审查**

#### 3.3.1 Plan Agent
- 接收用户复杂请求
- 分解为子任务列表
- 生成执行计划

#### 3.3.2 Execute Agent
- 执行计划中的每个子任务
- 使用工具完成具体工作
- 记录执行过程和结果

#### 3.3.3 Review Agent
- 审查执行结果
- 检查是否符合预期
- 提出改进建议

#### 3.3.4 工作流状态机
```
User Request → Plan → [Execute] → Review → Done
                    ↑_____________|
                    (循环直到通过审查)
```

#### 3.3.5 Web UI 展示
- 显示当前工作流状态
- 展示 Plan/Execute/Review 的进度
- 用户可以干预和修改计划

---

### Phase 4: Agent Teams（v0.14.0）
**目标：多个 Agent 并行协作**

#### 3.4.1 Agent Registry
- 动态注册和管理 Agent
- 每个 Agent 有独立身份和技能
- Agent 间可以互相调用

#### 3.4.2 任务分配
- Coordinator 分配任务给不同 Agent
- 支持并行执行
- 结果汇总和整合

#### 3.4.3 团队模式
```
Coordinator (架构师)
    ├── Developer A (前端)
    ├── Developer B (后端)
    └── Reviewer (代码审查)
```

---

### Phase 5: 代码理解（v0.15.0）
**目标：让 Agent 理解代码库**

#### 3.5.1 代码库索引
- 自动扫描项目文件结构
- 生成 codebase-index.md
- 记录关键模块和文件

#### 3.5.2 语义搜索
- 使用向量搜索找到相关代码
- 支持自然语言查询代码

#### 3.5.3 代码上下文注入
- 自动读取相关代码文件
- 注入到对话上下文中

## 四、技术实现要点

### 4.1 Memory 存储
- 文件系统：Markdown + Frontmatter
- 索引：MEMORY.md（200 行上限）
- 检索：基于文件名和描述的简单匹配

### 4.2 自动收集流程
```
对话结束 → LLM 总结 → 提取关键信息 → 分类 → 保存到 memory/
         → 更新 MEMORY.md 索引
```

### 4.3 记忆注入提示词
```
# Memory System

You have access to persistent memories stored in ~/.myagent/memory/.

## Current Memories
{memory_content}

## Guidelines
- Use memories to personalize your responses
- Verify memories against current state before acting
- Update or remove stale memories
```

### 4.4 Web UI 改造
- 新增 "Memory" 标签页
- 显示记忆列表和详情
- 支持编辑和删除
- 新增 "Workspace" 标签页
- 显示项目结构和文件树
